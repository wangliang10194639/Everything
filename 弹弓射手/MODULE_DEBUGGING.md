# 微信小游戏模块路径调试指南

## 🐛 当前问题

编译报错：
```
Error: module 'js/scenes/GameScene.js' is not defined, require args is './scenes/GameScene'
```

## 🔍 问题分析与解决方案演进

### 方案演进历程：

1. **带 `./` 前缀相对路径** (`./scenes/GameScene`)
   - 结果：微信添加 `js/` 前缀 → `js/scenes/GameScene.js` ❌ 仍有问题

2. **无前缀相对路径** (`scenes/GameScene`) ← **当前采用**
   - 预期：微信正确解析为 `js/scenes/GameScene.js` ✅

3. **绝对路径** (`/js/scenes/GameScene`)
   - 结果：被添加额外 `js/` 前缀 → `js/js/scenes/GameScene.js` ❌

## 🛠️ 最终推荐方案：无前缀相对路径

根据项目规范和实践经验，**优先使用不带 `./` 前缀的相对路径**：

**修改文件**：
- `game.js`: `import Main from 'js/main';`
- `js/main.js`: `import GameScene from 'scenes/GameScene';`

**核心优势**：
- 符合微信小游戏的自动路径解析机制
- 避免手动前缀导致的路径冲突
- 统一项目内的导入风格
