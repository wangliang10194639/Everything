游戏策划案：《弹弓奇兵：城堡冲刺》

1. 游戏概述

1.1 游戏类型

• 混合玩法：弹弓射击 + Roguelike元素 + 轻策略收集

• 目标平台：微信小游戏

• 核心体验：单指操作、快节奏闯关、随机Build构筑、满足收集欲

1.2 一句话简介

使用弹弓发射你的“奇兵”，摧毁沿途的邪恶城堡，收集上百种技能符文，组建独一无二的传奇队伍。

1.3 市场定位与用户分析

维度 说明

目标用户 25-35岁上班族、喜欢轻度策略和收集的玩家

游戏时长 单局3-5分钟，适合通勤、休息间隙

核心驱动力 “再来一局”的即时反馈 + “收集完整”的长期目标

竞争优势 操作简单但策略深度足；Roguelike元素保证每局新鲜感

2. 核心玩法

2.1 基础操作

1. 拖拽发射：按住屏幕上的奇兵角色，向后拖拽调整角度和力度，松开手指发射。
2. 抛物线物理：真实物理轨迹，可计算反弹、碰撞效果。
3. 技能触发：击中前划动屏幕可激活角色专属技能（如：分裂、爆破、穿墙）。

2.2 关卡设计

• 结构：每个关卡由3-5个连续战斗场景组成，最终面对城堡Boss。

• 进程：主地图采用“路线选择”模式（类似《杀戮尖塔》），玩家可选择不同路线（普通战斗/精英战斗/事件/商店）。

• 胜利条件：摧毁场景内所有敌方建筑或击败所有敌人。

• 失败条件：己方所有奇兵耗尽。

2.3 成长系统（单局Roguelike）

流程图示例
玩家开始游戏 → 选择初始奇兵 → 进入第一场战斗 → 获胜后三选一符文奖励 → 路线选择进入下一事件 → 累积符文组合成Build → 挑战关卡Boss

• 符文系统：每场战斗胜利后，随机出现3个符文供选择，分为：

  • 攻击类：暴击率+10%、穿透+1、爆炸范围扩大...

  • 防御/辅助类：击杀回血、护盾生成、金币获取增加...

  • 特殊效果类：连锁闪电、冰冻效果、召唤小兵...

• 符文协同：收集特定组合激活额外效果（如：集齐3个“火焰”符文，所有攻击附加灼烧）。

2.4 经济系统

• 战斗内金币：摧毁建筑或敌人掉落，用于单局内的临时商店购买符文或恢复。

• 全局货币（钻石）：通过每日任务、成就、看广告获取，用于解锁新角色和皮肤。

3. 主要系统

3.1 奇兵角色系统

角色 初始技能 解锁方式 定位

石头兵（初始） 重型冲击：对城墙造成双倍伤害 默认解锁 坦克、破墙

火焰精灵 烈焰附着：击中后持续灼烧5秒 通过第3关解锁 持续伤害

闪电球 连锁闪电：弹射3个额外目标 观看广告解锁一次试用 AOE清场

忍者镖 手里剑分身：击中后分裂为3个小飞镖 成就解锁 高频率攻击

治疗水滴 生命滋养：为周围友方单位恢复生命 内购或活动获取 辅助

3.2 外围成长系统

1. 天赋树：使用全局货币解锁永久被动加成（如：所有奇兵生命+5%、开局金币+20）。
2. 图鉴收集：每个符文、角色、敌人、Boss均有独立图鉴，收集完成提供全局属性加成。
3. 每日挑战：特殊规则关卡（如：只能用火焰精灵、重力减半），奖励丰厚。

3.3 社交与竞技

• 好友排行榜：比较每日挑战分数。

• 幽灵系统：可观看好友的闯关录像（以幽灵形态同步演示其操作）。

• 每周Boss竞赛：全服玩家共同挑战超高难度Boss，按伤害排名发放奖励。

4. 美术与音效

4.1 视觉风格

• 美术基调：欧美卡通休闲风，色彩明亮，造型Q萌略带搞怪。

• 场景设计：森林、沙漠、雪山、火山、云端城堡等主题关卡。

• 特效重点：弹弓拖拽时的力度指示线、击中时的粒子爆炸、符文激活时的炫光效果。

4.2 音效设计

• 背景音乐：轻快活泼的循环音乐，不同关卡有风格变奏。

• 操作音效：拖拽时的“皮筋”音效、发射时的“嗖”声、碰撞的“砰”声。

• 反馈音效：获得新符文时的“叮咚”清脆音、Boss出场震撼音。

5. 商业化设计

5.1 广告变现

• 激励视频广告：

  1. 复活续关：关卡失败后，可观看广告立即复活并恢复50%兵力。
  2. 符文重掷：不喜欢当前三个符文？看广告刷新一次。
  3. 双倍金币：关卡胜利后，可选择观看广告获得双倍金币奖励。
  4. 免费宝箱：每4小时可开启的免费宝箱，看广告可直接开启免等待。
  5. 试用新英雄：新英雄可免费试用一局，后续想再次试用需看广告。
• 插屏广告：

  • 在关卡结束返回地图界面时，有概率展示（每日前3局不展示）。

• Banner广告（可选）：

  • 在游戏主界面底部固定展示，设计成公告栏样式减少违和感。

5.2 内购设计（虚拟道具，无需版号）

• 永久去广告：一次性付费移除所有插屏广告和Banner广告（激励视频保留）。

• 特权月卡：

  • 每日登录领取钻石和特殊符文。

  • 天赋树经验获取+50%。

  • 专属头像框和聊天气泡。

• 直接购买：钻石礼包、限定皮肤（纯外观）、新手成长礼包。

6. 开发与运营计划

6.1 开发阶段

阶段 周期 核心目标 内容范围

MVP版本 4-6周 验证核心玩法 1个基础角色、10个关卡、30种符文、基础广告接入

EA版本 3-4周 数据调优与留存 新增2个角色、天赋树系统、每日挑战、社交排行榜

正式版本 4周 商业化与内容扩充 完整5个角色、3大章节（共45关）、月卡与皮肤系统

6.2 数据监控重点

• 核心指标：次留（>40%）、七留（>15%）、平均游戏时长（>15分钟/日）、eCPM。

• 玩法验证：符文选择分布、最受欢迎角色、关卡难度曲线（通过率）。

• 商业化数据：激励视频展示率/完成率、人均广告次数、付费转化率。

6.3 长期内容规划

1. 赛季更新：每8-12周为一个赛季，推出新主题章节、新Boss、新符文系列。
2. 特殊活动：节假日限定活动（如春节年兽Boss）、联动活动。
3. UGC可能性：后期可开放“自定义关卡编辑器”，让玩家设计并分享关卡。

7. 开发备忘录

• 技术选型：建议使用Cocos Creator开发，对2D物理和微信小游戏平台支持良好。

• 性能优化：注意控制同屏粒子特效数量，避免低端机卡顿。

• 合规注意：广告频率需符合微信小游戏规范，避免用户反感；虚拟道具购买需明确提示。

• 测试重点：物理碰撞手感、符文组合平衡性、新手引导流畅度。

游戏卖点总结：  
1. 极易上手：单指拖拽操作，零学习成本。  
2. 重复可玩性高：Roguelike符文系统保证每局构筑不同。  
3. 广告场景自然：失败复活、奖励加倍等均为玩家主动选择。  
4. 长期目标清晰：收集图鉴、培养天赋、解锁角色提供持续动力。  
5. 开发风险可控：MVP版本只需核心玩法验证，可快速迭代。

此策划案聚焦于微信小游戏用户“短平快”的体验需求，同时通过深度构筑和收集系统增强留存，商业模型经过市场验证，适合中小团队开发。
